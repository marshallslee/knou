# SDN 및 P4 기반의 무손실 이동 표적 방어(MTD) 메커니즘에 관한 연구

## A Study on Lossless Moving Target Defense (MTD) Mechanism Based on SDN and P4

이세영

Seyoung Lee

---

### 요 약

본 연구는 현대 네트워크 보안의 고질적인 문제인 정적 구성 취약성을 해결하기 위해 소프트웨어 정의 네트워킹 (SDN)과 P4를 결합한 무손실 이동 표적 방어 (MTD) 메커니즘을 제안한다. MTD는 시스템의 공격 표면을 동적으로 변화시켜 공격자의 정찰, 무기화, 침투 등 사이버 킬 체인의 초기 단계를 방해하는 능동적 방어 전략이다. 기존 SDN 기반 MTD 기법들은 컨트롤 평면에서 주소 변이를 처리하여 컨트롤러 병목, 패킷 유실, TCP 세션 단절 등의 문제를 야기하였다. 이를 해결하기 위해 P4 프로그래밍 가능 데이터 평면에서 실시간 주소 변이를 수행하는 아키텍처를 설계하고, 이중 매핑 테이블과 플로우 추적 기반의 점진적 전환 알고리즘을 통해 무손실 연결을 보장한다. 경계 스위치에서 가상 IP와 실제 IP를 동적으로 매핑함으로써 외부 공격자에게는 혼란을 주면서도 내부 통신의 연속성을 유지한다. 현재 Mininet과 BMv2 기반 테스트베드 구축을 완료하고 P4 데이터 평면의 핵심 로직 구현을 진행 중이며, 향후 보안성과 네트워크 오버헤드에 대한 정량적 평가를 수행할 예정이다.

**주제어:** 이동 표적 방어 (MTD), 소프트웨어 정의 네트워킹(SDN), P4, 무손실 주소 변이, 네트워크 보안, 데이터 평면

### ABSTRACT

This study proposes a lossless Moving Target Defense (MTD) mechanism that combines Software-Defined Networking (SDN) and P4 to address the inherent vulnerability of static network configurations in modern network security. Existing SDN-based MTD techniques suffer from controller bottlenecks, packet loss, and TCP session disruption during address mutation, as they handle address translation at the control plane. To overcome these limitations, we design an architecture that performs real-time address mutation at the P4-programmable data plane and ensures lossless connectivity through a dual mapping table and flow-tracking-based gradual transition algorithm. By dynamically mapping virtual IPs to real IPs at edge switches, the proposed mechanism confuses external attackers while maintaining continuity of internal communications. We have completed the construction of a Mininet and BMv2-based testbed and are currently implementing the core P4 data plane logic. Quantitative evaluation of security effectiveness and network overhead will be conducted in subsequent phases.

**Keywords:** Moving Target Defense (MTD), Software-Defined Networking (SDN), P4, Lossless Address Mutation, Network Security, Data Plane

## 1. 서론

### 1.1 연구 배경 및 동기

현대 네트워크는 IP 주소, 포트, 라우팅 경로 등 주요 자원이 고정된 정적 구조로 운영되고 있다. 이러한 구조는 공격자가 충분한 정찰 시간을 확보할 수 있게 하며, 한 번 발견된 취약점이 지속적으로 공격에 활용되는 문제를 발생시킨다. 쉽게 말해 방어자는 모든 곳을 다 막아야 하는데, 공격자는 하나만 뚫으면 되는 구조인 셈이다. 기존 보안 시스템은 방화벽, 침입 방지 시스템 (IPS), 네트워크 접근 제어 (NAC) 등 수동적 방어 체계에 의존해 왔으나, Zero-day 공격이나 APT와 같은 지능형 공격에 효과적으로 대응하기 어렵다.

이러한 비대칭적 공방 구조를 해결하기 위해 이동 표적 방어 (Moving Target Defense, MTD) 기술이 주목받고 있다. MTD는 시스템의 공격 표면을 동적으로 변화시켜 공격자의 정찰 활동을 무력화하는 능동적 방어 전략이다. 그러나 MTD 기술, 특히 네트워크 주소변이 기술을 기존의 고정 IP 기반 보안 체계에 적용할 경우, 방화벽, NAC, IPS 등 주요 보안 시스템과의 상호운용성 문제가 발생할 수 있어 실제 적용에는 여전히 과제가 남아 있다.

한편, P4 (Programming Protocol-Independent Packet Processors)의 등장은 네트워크 데이터 평면의 프로그래밍 가능성을 혁신적으로 확장하였다. P4는 프로토콜에 독립적인 패킷 프로세서를 프로그래밍하기 위한 고수준 언어로, OpenFlow가 고정된 프로토콜 헤더에 의존하여 복잡성이 증가하는 문제를 해결하기 위해 제안되었다. P4를 통해 스위치 수준에서 사용자 정의 패킷 파싱, 헤더 조작, 매치-액션 처리를 직접 수행할 수 있게 되면서, 기존에 컨트롤러에 의존하던 복잡한 네트워크 기능을 데이터 평면으로 오프로드 (offload)할 수 있게 되었다. 이는 MTD의 핵심 동작인 주소 변이를 와이어 속도 (wire speed)로 처리할 수 있는 가능성을 열어주었다.

### 1.2 연구 목적 및 범위

본 연구의 목적은 SDN과 P4를 결합하여 보안성 강화와 가용성 유지 사이의 트레이드오프 문제를 해결하는 무손실 MTD 메커니즘을 설계하고 그 효과를 검증하는 것이다. 구체적인 연구 목표는 다음과 같다.

첫째, P4 프로그래밍을 활용하여 경계 스위치 (edge switch)의 데이터 평면에서 직접 가상 IP-실제 IP 매핑을 수행함으로써, SDN 컨트롤러의 개입을 최소화하고 주소 변이에 따른 지연을 마이크로초 수준으로 줄이는 아키텍처를 설계한다.

둘째, 주소 변이 과정에서 기존 TCP 세션의 연속성을 보장하는 무손실 전환 알고리즘을 제안한다. 이를 위해 플로우 상태 추적 메커니즘과 이중 매핑 테이블 (dual mapping table)을 활용한 점진적 전환 방식을 도입한다.

셋째, Mininet과 BMv2 (Behavioral Model version 2) 소프트웨어 스위치를 활용한 프로토타입 실험 환경을 구축하여, 제안 메커니즘의 보안성 (스캔 공격 차단율)과 성능 (지연 시간, 처리량, 패킷 유실률)을 정량적으로 평가한다.

## 2. 관련 연구

### 2.1 MTD 기술 동향 파악

먼저, MTD 최신 기술 동향이 파악하다는 판단하에 관련 논문을 몇 찾아서 읽는 작업을 수행하였다. 이수원 등[2]은 네트워크 주소변이 기술을 기존 보안 체계에 적용했을 때의 영향성을 분석하며, 방화벽, NAC, IPS 등 주요 보안 시스템과의 상호운용성을 구체적으로 검토하였다. 이 연구에서는 IPS의 경우 시그니처 기반 탐지 방식이 페이로드 분석 위주이므로 주소변이에 의한 영향이 미미하지만, 방화벽의 경우 변이되는 주소 대역 전체를 정책에 반영해야 하는 과제가 있음을 도출하였다. 또한 변이 전/후 주소를 매핑할 수 있는 대표 IP 개념과 시간별 변이 이력을 결합한 통합 관리 모델을 제시하여, 주소가 계속 바뀌는 환경에서도 네트워크 분석 시스템이 공격자를 추적할 수 있는 방안을 마련하였다[2].

Jalowski 등[1]은 네트워크 MTD에 대한 실용적 관점의 서베이를 수행하면서, MTD 기법을 적용 대상(what to move), 동작 방식(how to move), 적용 시점(when to move)의 세 가지 기준으로 체계적으로 분류하였다. IP 주소 및 포트 변경, 경로(route) 변경, 호스트 변경 등 다양한 변이 대상을 분석하였으며, 대부분의 연구에서 랜덤 방식을 채택하고 있음을 확인하였다. 또한 정찰 공격, DoS/DDoS, Zero-day, APT 등 다양한 공격 시나리오에 대한 각 MTD 기법의 방어 효과를 비교 분석하면서, 실무 적용 관점에서 Hybrid MTD, 성능 최적화, 실제 환경 검증의 필요성을 제시하였다[1]. 다만 이 서베이에서는 대부분의 연구가 시뮬레이션 기반으로 이루어져 실제 대규모 네트워크 환경에서의 검증이 부족한 점이 한계로 지적되었다.

Hyder와 Fatima[5]는 SDN 환경에서 지능형 DDoS 공격인 크로스파이어 공격을 방어하기 위해 IBN(의도 기반 네트워킹)과 MTD를 결합한 프레임워크를 제안하였다. 크로스파이어 공격은 직접적인 타겟 대신 타겟 주변의 링크를 범람시켜 네트워크를 마비시키는 특성 때문에 탐지가 매우 어려운데, 본 연구를 진행하면서 이러한 유형의 공격이 존재한다는 것 자체가 상당히 흥미로웠다. 이 연구에서는 ONOS SDN 컨트롤러의 REST API와 DNS 포트 리다이렉션을 활용해 실제 호스트로 향하는 트래픽을 가상 섀도우 호스트(virtual shadow hosts)로 분산시킴으로써, 공격자가 네트워크 링크의 연결 상태를 파악하지 못하게 하는 동적 방어 체계를 구축하였다[5]. 대규모 네트워크 자원을 소모하지 않고도 DNS와 REST API만을 활용해 저비용-고효율의 보안 솔루션을 구현하였으며, Mininet 에뮬레이터와 ONOS 컨트롤러를 사용하여 성능을 검증하였다. 다만 빈번한 DNS 리다이렉션과 경로 변경이 정상 사용자의 latency에 미치는 영향에 대한 심층 분석이 다소 부족하고, 공격 규모가 매우 클 경우 가상 섀도우 호스트 자체가 자원 고갈 상태에 빠질 수 있는 2차 DDoS 가능성에 대한 고려가 필요해 보인다[5].

### 2.2 SDN 및 P4 프로그래밍의 이해

소프트웨어 정의 네트워킹(SDN)은 네트워크의 컨트롤 평면과 데이터 평면을 분리하여 중앙 집중적인 네트워크 제어를 가능하게 하는 아키텍처이다[4][5]. 조부승[4]은 SDN 기술을 통해 네트워크의 제어평면과 데이터평면으로 네트워크 기능을 분리하여 구현함으로써, 네트워크 구성 요소에 대한 중앙 집중적인 네트워크 제어를 가능하게 한다고 설명하였다. Hyder와 Fatima[5] 역시 SDN이 네트워크 제어와 데이터 평면을 분리하여 유연한 관리를 가능케 하는 핵심 기술임을 전제로 연구를 수행하였다.

그러나 OpenFlow는 고정된 프로토콜 헤더 집합에 의존하여 초기 12개 필드에서 41개 이상으로 급증하는 복잡성 문제를 야기하였으며, 새로운 프로토콜 헤더를 추가하는 유연성이 부족하였다[3]. 이를 해결하기 위해 Bosshart 등[3]이 제안한 P4는 프로토콜에 독립적인 패킷 프로세서를 프로그래밍하기 위한 고수준 도메인 특화 언어(DSL)이다. P4의 핵심 구성 요소는 헤더 정의(header definition), 파서(parser), 매치-액션 테이블(match-action table), 액션(action), 제어 프로그램(control program)으로, 사용자가 패킷의 파싱 방법과 처리 로직을 직접 정의할 수 있다[3].

P4의 핵심 목표는 세 가지이다. 첫째, 현장 재구성 가능성(reconfigurability)으로, 네트워크 운영 중에도 패킷 처리 로직을 변경할 수 있다. 둘째, 프로토콜 독립성(protocol independence)으로, 특정 프로토콜에 종속되지 않는 범용적 패킷 처리가 가능하다. 셋째, 타겟 독립성(target independence)으로, 동일한 P4 프로그램이 소프트웨어 스위치, FPGA, ASIC 등 다양한 타겟에서 실행될 수 있다[3]. P4의 추상 포워딩 모델은 프로그래밍 가능한 파서와 직렬/병렬로 구성된 매치-액션 파이프라인으로 구성되며, P4 컴파일러가 프로그램을 테이블 의존성 그래프(Table Dependency Graph)로 변환한 후 타겟 하드웨어에 매핑한다[3].

P4의 보안 분야 활용에 대한 연구도 활발히 진행되고 있다. 조부승[4]은 P4 데이터 평면 프로그래밍을 통해 네트워크 내 모든 플로우를 P4 스위치 단에서 실시간으로 모니터링하고, SDN 제어기와 연동하여 DDoS 공격 및 IP 스푸핑 공격에 대응하는 정책 기반 인-네트워크(In-Network) 보안 관리 방법을 제안하였다. 조부승은 기존의 OpenFlow 기반 SDN 스위치에서는 플로우 혹은 세션의 상태 모니터링이 불가하여 동적으로 전개되는 사이버 공격에 대한 대응이 힘든 반면, P4 스위치에서는 모든 네트워크 세션의 상태를 모니터링하고 추적할 수 있어 네트워크 단에서의 즉각적인 보안 공격 탐지 및 대응이 가능함을 보였다[4]. 이는 기존 SDN 제어기 기반 보안의 지연 시간 문제를 데이터 평면 수준에서 해결할 수 있음을 보여준 사례이다.

김수연과 박태준[6]은 P4 기반 방화벽을 구현하여, 패킷 필터링과 같은 전통적 보안 기능을 프로그래밍 가능한 데이터 평면에서 직접 수행할 수 있음을 제시하였다. P4 기반 방화벽은 소프트웨어의 성능 문제와 하드웨어의 벤더 종속성을 모두 해결할 수 있는 장점이 있으며, 새로운 취약점이 발견될 경우 프로그래머가 필요한 보안 규칙을 직접 프로그래밍하고 배포함으로써 대응 시간을 단축할 수 있다[6]. 다만 이 논문에서도 아직 P4 기반 방화벽이 가지고 있는 몇 가지 문제점, 예를 들면 DNS 패킷에만 국한되는 DPI 시스템의 한계 같은 부분이 해결되지 않은 채로 남아있었다[6]. 그럼에도 불구하고 이들 연구는 P4 데이터 평면에서 보안 기능을 구현하는 것이 충분히 가능하다는 점을 보여주고 있어, 본 연구에서 P4를 활용한 주소 변이 구현의 기술적 타당성을 뒷받침해준다고 생각한다.

### 2.3 기존 MTD 기술의 성능 한계 분석

기존 SDN 기반 MTD 기법들의 주요 한계는 크게 세 가지로 분류할 수 있다.

첫째, 컨트롤러 병목 문제이다. 조부승[4]이 지적한 바와 같이, 기존의 OpenFlow 기반 SDN 스위치의 데이터 평면으로 보안 위협 패킷이 유입될 경우 SDN 제어기에 처리 방법을 요청한 후 해당 결과를 다시 SDN 스위치에 내려보내야 하므로 물리적인 통신 지연 시간 및 처리 시간이 소요되어 즉각적인 대응이 불가능하다[4]. 이 문제는 주소 변이를 SDN 컨트롤러에서 처리하는 MTD 기법에서 더욱 심각해진다.

둘째, 패킷 유실 문제이다. 주소 변이가 발생하는 순간, 이전 매핑 정보로 전송 중이던 패킷이 새로운 플로우 규칙과 불일치하여 드롭되는 문제가 발생한다. 특히 TCP와 같은 연결 지향 프로토콜에서는 주소 변이 시 세션이 단절되어 재연결이 필요하며, 이는 실시간 서비스 환경에서 심각한 QoS 저하를 초래한다. 이 문제는 사실 본 연구의 출발점이 된 핵심적인 고민이었다. 이수원 등[2]의 연구에서도 주소변이 기술 적용 시 기존 보안 장비의 정책 변화 및 영향성을 분석하면서, 변이 과정에서의 서비스 중단이나 보안 정책 무력화가 우려됨을 지적한 바 있다[2].

셋째, 플로우 테이블 확장성 문제이다. MTD의 주소 변이 주기가 짧아질수록 SDN 스위치의 플로우 테이블에 설치해야 하는 규칙 수가 급증한다. 조부승[4]은 P4 기반 Tofino 칩의 경우 O(10MB) 수준의 SRAM만 장착되어 있어 수백만 개 이상의 플로우 정보를 모두 저장하는 것이 불가능함을 지적하며, 이를 보완하기 위한 외부 메모리 활용 방안의 필요성을 논의하였다[4].

이러한 한계를 극복하기 위해 P4 기반 데이터 평면에서 직접 주소 변이를 수행하는 접근법이 주목받고 있다. 본 연구는 P4 데이터 평면의 상태 유지(stateful) 기능을 활용한 플로우 추적 메커니즘과 이중 매핑 테이블을 통한 점진적 전환 알고리즘을 도입하여 무손실 보장을 제공한다.

## 3. 연구 방법

### 3.1 시스템 아키텍처 및 구성 요소

제안하는 시스템 아키텍처는 크게 세 가지 구성 요소로 이루어진다: SDN 컨트롤러, P4 경계 스위치(edge switch), 그리고 P4 코어 스위치(core switch)이다.

SDN 컨트롤러는 전체 MTD 정책을 관리하는 중앙 제어 장치로서, 변이 주기(mutation interval) 결정, 가상 IP 풀(vIP pool) 관리, vIP-rIP 매핑 테이블 생성 등의 역할을 수행한다. 초기 설계 단계에서는 컨트롤러가 모든 주소 변환을 직접 처리하는 방식도 고려하였으나, 조부승[4]이 지적한 SDN 제어기의 지연 시간 문제를 감안하여 이 방식을 폐기하였다. 조부승[4]이 제안한 정책 기반 인-네트워크 보안 관리 구조와 유사하게, SDN 제어기는 보안 정책을 반영하는 역할만을 수행하며 실제 패킷별 주소 변환은 P4 스위치에서 자율적으로 수행된다. 이를 통해 컨트롤러의 부하를 대폭 감소시킨다. 조부승[4]의 연구에서는 SDN 제어기를 통해 블랙리스트 혹은 화이트리스트를 업데이트하여 네트워크 내 P4 스위치를 항상 최신의 보안 정책으로 운영하는 구조를 제시하였는데, 본 연구에서는 이 개념을 MTD 매핑 테이블 관리에 확장 적용한다.

P4 경계 스위치는 내부 네트워크와 외부 네트워크의 접점에 위치하며, MTD의 핵심 동작인 주소 변환을 수행한다. 외부에서 유입되는 패킷에 대해서는 목적지의 가상 IP를 실제 IP로 변환(ingress translation)하고, 내부에서 외부로 나가는 패킷에 대해서는 출발지의 실제 IP를 가상 IP로 변환(egress translation)한다. P4 프로그래밍을 통해 이 과정이 데이터 평면에서 와이어 속도로 처리된다[3][4].

P4 코어 스위치는 내부 네트워크의 패킷 포워딩을 담당하며, 실제 IP 기반의 라우팅을 수행한다. 이수원 등[2]이 제안한 대표 IP 개념과 유사하게, 내부 호스트 간 통신은 실제 IP를 사용하므로 MTD의 주소 변이에 영향을 받지 않아 내부 서비스의 안정성을 보장한다.

### 3.2 P4 기반 데이터 평면 설계

P4 경계 스위치의 데이터 평면은 다음과 같은 파이프라인으로 구성된다.

**파서(Parser)** 단계에서는 이더넷 헤더, IPv4 헤더, TCP/UDP 헤더를 순차적으로 파싱하여 패킷의 주요 필드를 추출한다. Bosshart 등[3]이 정의한 P4의 파서는 패킷 내에서 헤더 시퀀스를 식별하고 추출하는 상태 머신으로, P4의 프로토콜 독립성을 활용하여 필요 시 사용자 정의 메타데이터 헤더를 추가로 정의할 수 있다.

**인그레스 파이프라인(Ingress Pipeline)** 에서는 다음의 매치-액션 테이블들이 순차적으로 적용된다:

1) vIP-to-rIP 변환 테이블: 외부에서 유입되는 패킷의 목적지 가상 IP를 실제 IP로 변환한다. 이 테이블은 SDN 컨트롤러로부터 주기적으로 업데이트되며, 현재 활성화된 매핑과 이전 매핑을 동시에 유지하는 이중 매핑 구조를 가진다.

2) 플로우 상태 테이블: P4 레지스터(register)를 활용하여 활성 플로우의 5-tuple(출발지 IP, 목적지 IP, 출발지 포트, 목적지 포트, 프로토콜) 정보와 해당 플로우가 사용하는 매핑 버전을 기록한다. 조부승[4]이 제안한 바와 같이, P4 스위치 내 SRAM 레지스터를 통해 패킷 당 라인 속도로 플로우 정보를 모니터링하고 최신 상태를 유지할 수 있다[4].

3) 포워딩 테이블: 변환된 실제 IP를 기반으로 적절한 출력 포트로 패킷을 전달한다.

**이그레스 파이프라인(Egress Pipeline)** 에서는 rIP-to-vIP 변환 테이블을 통해 내부에서 외부로 나가는 패킷의 출발지 실제 IP를 현재 활성화된 가상 IP로 변환한다. 이때 응답 패킷의 경우 해당 플로우의 매핑 버전을 참조하여 요청 시 사용된 가상 IP와 일치하는 주소를 사용한다.

**디파서(Deparser)** 단계에서는 수정된 헤더 필드를 다시 패킷에 조립하고, IPv4 체크섬을 재계산한다. P4의 체크섬 업데이트 기능을 활용하여 증분 체크섬 계산(incremental checksum update)을 수행함으로써 처리 오버헤드를 최소화한다[3].

### 3.3 무손실 IP 변이 알고리즘

무손실 주소 변이의 핵심은 매핑 전환 시 기존 활성 플로우의 연속성을 보장하는 것이다. 이를 위해 본 연구에서는 이중 매핑 테이블과 플로우 추적 기반의 점진적 전환 알고리즘을 제안한다.

**단계 1 - 새 매핑 준비(Prepare Phase):** SDN 컨트롤러가 새로운 vIP-rIP 매핑을 생성하고, P4 스위치의 이중 매핑 테이블 중 비활성 슬롯에 새 매핑을 설치한다. 이 단계에서 기존 트래픽은 현재 활성 매핑을 계속 사용한다.

**단계 2 - 병행 운영(Coexistence Phase):** 새로운 매핑이 활성화되어 새로 수립되는 연결에 적용된다. 동시에 기존 활성 플로우에 대해서는 플로우 상태 테이블을 참조하여 이전 매핑을 계속 적용한다. P4 레지스터에 기록된 각 플로우의 매핑 버전 정보를 통해 패킷별로 적절한 매핑을 선택한다. 조부승[4]이 제시한 P4 스위치의 SRAM 레지스터 기반 플로우 모니터링 메커니즘을 활용하여, 각 플로우의 상태를 패킷 당 라인 속도로 추적한다.

**단계 3 - 정리(Cleanup Phase):** 이전 매핑을 사용하는 모든 활성 플로우가 종료되거나 타임아웃되면, 이전 매핑 슬롯을 해제한다. P4 카운터(counter)를 활용하여 이전 매핑을 참조하는 플로우 수를 모니터링하며, 카운터가 0이 되면 컨트롤러에 완료를 통보한다.

이 알고리즘의 핵심 장점은 매핑 전환이 원자적(atomic) 연산이 아닌 점진적(gradual) 전환으로 이루어지므로, 어떤 시점에서도 활성 플로우의 패킷이 매핑 불일치로 드롭되지 않는다는 것이다. 물론 이론적으로는 그렇지만, 실제 구현에서는 레지스터 경합(race condition)이나 타이밍 이슈가 발생할 가능성도 있어서 이 부분은 구현 단계에서 좀 더 면밀히 검증해 볼 필요가 있다.

### 3.4 실험 환경 구축

제안 메커니즘의 효과를 검증하기 위해 Mininet 네트워크 에뮬레이터와 BMv2(Behavioral Model version 2) P4 소프트웨어 스위치를 기반으로 실험 환경을 구축하였다. 실험 환경 선정에 있어서 처음에는 하드웨어 P4 스위치를 사용하는 방안도 검토하였으나, 비용과 접근성 문제로 인해 소프트웨어 기반 환경을 선택하게 되었다. Hyder와 Fatima[5]도 크로스파이어 공격 방어 연구에서 Mininet 에뮬레이터와 ONOS 컨트롤러를 사용하여 SDN 기반 MTD의 성능을 검증한 바 있어, 소프트웨어 환경에서의 실험도 충분한 의미가 있다고 판단하였다. 이승운과 노병희[7]는 SDN 모델링 및 시뮬레이션을 위한 Mininet과 OPNET의 성능을 비교 분석하여 Mininet이 처리량 측면에서는 실제 환경을 잘 반영하나, 지연 시간이나 지터와 같은 세부 성능 지표의 정밀도에는 한계가 있음을 보고하였다[7]. 본 연구에서는 Mininet의 처리량 반영 강점을 활용하되, 지연 시간 측정 시 이러한 제한 사항을 고려하여 결과를 해석할 예정이다.

실험 토폴로지는 외부 네트워크(공격자 호스트 포함), P4 경계 스위치 2대, P4 코어 스위치 1대, 내부 네트워크(서버 호스트 4대)로 구성된다. SDN 컨트롤러로는 ONOS 또는 Ryu를 사용하며, P4Runtime API를 통해 컨트롤러와 P4 스위치 간의 통신을 수행한다.

실험 파라미터는 다음과 같이 설정한다: 가상 IP 풀 크기는 /16 서브넷(65,536개 주소), 변이 주기는 5초에서 60초까지 가변, TCP 연결은 iperf3를 통한 장기 세션으로 구성한다. 비교 대상으로는 MTD 미적용 환경, OpenFlow 기반 컨트롤러 MTD, 그리고 제안하는 P4 기반 무손실 MTD를 설정한다.

현재까지 Mininet 및 BMv2 기반 테스트베드 구축을 완료하고 기본 네트워크 토폴로지의 연결성 검증을 마쳤으며, P4 데이터 평면의 핵심 로직 구현을 진행 중인 단계이다.

| 마무리 | 최종 데이터 정리 및 보고서 작성 | 13~15주차 | 예정 |

1~2주차에는 MTD 기술 동향, SDN/P4 프로그래밍, 기존 MTD의 성능 한계에 관한 체계적인 문헌 조사를 수행하였다. 이 과정에서 Jalowski 등[1]의 MTD 서베이, Bosshart 등[3]의 P4 프로그래밍 기초, 조부승[4]의 P4 기반 보안 관리 등 핵심 문헌을 분석하여 연구의 이론적 기반을 확립하였다.

2~4주차에는 SDN 컨트롤러, P4 경계 스위치, P4 코어 스위치로 구성된 3계층 시스템 아키텍처를 설계하고, P4 데이터 평면의 파이프라인 구조(파서, 인그레스, 이그레스, 디파서)를 상세 설계하였다. 또한 이중 매핑 테이블과 플로우 추적 기반의 무손실 IP 변이 알고리즘을 고안하였다.

3~6주차에는 Mininet 에뮬레이터와 BMv2 소프트웨어 스위치를 활용하여 실험 토폴로지를 구축하고, P4Runtime API를 통한 컨트롤러-스위치 간 통신 경로를 확보하였다. 기본 L2/L3 포워딩의 정상 동작을 검증하였다.

5주차부터는 구현 단계에 진입하여 P4 기반 주소 변이 핵심 로직의 구현을 진행 중이다. 현재 인그레스 파이프라인의 vIP-to-rIP 변환 테이블과 이그레스 파이프라인의 rIP-to-vIP 변환 테이블의 기본 동작을 구현하였으며, 이중 매핑 테이블과 플로우 상태 추적 레지스터의 통합 구현을 진행하고 있다. 솔직히 말하면 P4 문법에 익숙해지는 데 예상보다 시간이 많이 소요되었고, BMv2의 디버깅 과정에서 시행착오가 있어 일정이 당초 계획보다 다소 지연된 부분이 있다.

### 4.2 예상되는 보안성 분석

향후 실험에서는 Nmap을 이용한 네트워크 스캔 공격 시나리오를 구성하여 MTD의 보안 효과를 정량적으로 평가할 예정이다. 공격자는 외부 네트워크에서 대상 서브넷에 대해 TCP SYN 스캔, UDP 스캔, 핑 스윕을 수행하며, MTD 적용 시 실제 서비스 호스트의 탐지 성공률을 측정한다.

MTD의 보안 효과는 공격자의 정찰 성공 확률로 정량화할 수 있다. Jalowski 등[1]은 MTD 기법이 정찰 공격, DoS/DDoS, Zero-day, APT 등 다양한 공격 시나리오에 대해 방어 효과를 가짐을 비교 분석하였으며, 특히 IP 주소 및 포트 변경을 통한 MTD가 정찰 공격에 대해 가장 직접적인 방어 효과를 가짐을 확인하였다[1]. 가상 IP 풀의 크기를 N, 실제 호스트 수를 M, 변이 주기를 T, 공격자의 스캔 속도를 S(패킷/초)라 할 때, 공격자가 변이 주기 내에 특정 호스트를 정확히 탐지할 확률은 다음과 같이 근사할 수 있다:

> P_detect = 1 - (1 - M/N)^(S·T)

본 연구의 설정(N=65,536, M=4)에서 변이 주기 T가 짧아질수록 탐지 확률이 지수 함수적으로 감소할 것으로 예상된다. 예를 들어, 공격자의 스캔 속도가 1,000 패킷/초이고 변이 주기가 10초인 경우, 단일 변이 주기 내 특정 호스트 탐지 확률은 약 45.6%이지만, 변이 주기를 5초로 단축하면 약 26.3%로 감소한다.

### 4.3 예상되는 네트워크 오버헤드 분석

네트워크 성능 평가를 위해 다음의 지표를 측정할 예정이다.

**지연 시간(Latency):** P4 데이터 평면에서의 주소 변환에 소요되는 추가 지연을 측정한다. 조부승[4]이 지적한 바와 같이 P4 매치-액션 파이프라인의 각 스테이지는 ALU(Arithmetic Logic Unit)로 구현되어 하드웨어 속도로 헤더 필드 처리가 가능하므로, 추가 지연은 마이크로초 수준으로 예상된다[4]. 기존 OpenFlow 기반 MTD의 경우 컨트롤러 왕복 시간(RTT)이 추가되어 밀리초 수준의 지연이 발생하는 것과 대비된다.

**처리량(Throughput):** iperf3를 통해 MTD 적용 전후의 TCP/UDP 처리량을 측정한다. 이승운과 노병희[7]의 연구에서 Mininet이 처리량 측면에서 실제 환경을 잘 반영함을 확인하였으므로, 처리량 측정 결과의 신뢰성을 확보할 수 있을 것으로 판단된다[7]. P4 스위치의 매치-액션 처리는 기본 포워딩과 유사한 파이프라인 구조를 가지므로, 처리량 저하는 최소화될 것으로 예상된다.

**패킷 유실률(Packet Loss Rate):** 주소 변이 시점에서의 패킷 유실 여부를 정밀 측정한다. 제안하는 이중 매핑 테이블과 플로우 추적 메커니즘에 의해 패킷 유실률이 0%에 근접할 것으로 예상되며, 이는 기존 OpenFlow 기반 MTD에서 변이 시점마다 발생하던 패킷 유실과 명확히 대비된다.

**세션 연속성:** 장시간 TCP 세션(HTTP, SSH 등)이 주소 변이 과정에서 중단 없이 유지되는지를 확인한다. 웹 서버에 대한 장기 HTTP 다운로드와 SSH 세션을 유지한 상태에서 다수의 주소 변이를 수행하며 세션 단절 횟수를 기록할 예정이다.

## 5. 결론

본 연구에서는 SDN과 P4를 결합한 무손실 이동 표적 방어(MTD) 메커니즘을 제안하고, 이를 위한 시스템 아키텍처 설계 및 실험 환경 구축을 수행하였다. 기존 SDN 기반 MTD의 핵심 한계인 컨트롤러 병목, 패킷 유실, 세션 단절 문제를 P4 프로그래밍 가능 데이터 평면을 활용하여 해결하는 접근법을 제시하였다. 특히 이중 매핑 테이블과 플로우 상태 추적 기반의 점진적 전환 알고리즘을 통해 주소 변이 시점에서도 기존 활성 플로우의 연속성을 보장하는 무손실 메커니즘을 설계하였다.

현재까지의 연구 성과로는 첫째, 체계적인 문헌 분석을 통해 기존 MTD 기술의 한계와 P4 기반 접근법의 타당성을 확인하였다. 이수원 등[2]의 MTD 영향성 분석, Jalowski 등[1]의 MTD 서베이, Hyder와 Fatima[5]의 SDN 기반 MTD 프레임워크, Bosshart 등[3]의 P4 언어, 조부승[4]의 인-네트워크 보안 관리, 김수연과 박태준[6]의 P4 기반 방화벽 등의 연구를 종합적으로 분석하여 본 연구의 이론적 기반을 확립하였다. 둘째, SDN 컨트롤러, P4 경계 스위치, P4 코어 스위치로 구성된 3계층 아키텍처와 무손실 IP 변이 알고리즘을 설계하였다. 셋째, Mininet과 BMv2 기반 테스트베드를 구축하고 P4 데이터 평면의 핵심 변환 로직 구현에 착수하였다.

향후 연구 일정으로는 8~9주차에 P4 기반 주소 변이 및 무손실 로직 구현을 완료하고, 8~11주차에 Nmap 기반 스캔 공격 시뮬레이션과 iperf3 기반 가용성 측정을 수행할 예정이다. 10~13주차에는 네트워크 오버헤드 및 보안성 성능을 정량적으로 평가하며, 13~15주차에 최종 데이터 정리 및 연구 보고서를 완성할 계획이다.

장기적으로는 하드웨어 P4 스위치(예: Intel Tofino)에서의 실제 환경 검증이 가장 시급한 과제라고 생각된다. 소프트웨어 스위치 환경에서 아무리 좋은 결과가 나오더라도 실제 하드웨어에서의 검증 없이는 실용성을 주장하기 어렵기 때문이다. 그 외에도 강화학습 기반의 적응적 변이 주기 최적화, 포트 호핑 및 라우팅 경로 변이를 포함하는 다중 차원 MTD 통합, 그리고 이수원 등[2]이 제안한 변이 이력 기반 데이터 매핑을 활용한 통합 모니터링 체계 구축이 요구된다. 본 연구의 결과는 고신뢰 전술 네트워크, 기업용 클라우드 환경, 그리고 5G/6G 네트워크 슬라이싱 환경에서의 능동적 보안 체계 구축에 기여할 수 있을 것으로 기대된다.
