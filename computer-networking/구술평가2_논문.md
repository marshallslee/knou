SDN 및 P4 기반의 무손실 이동 표적 방어(MTD) 메커니즘에 관한 연구

A Study on Lossless Moving Target Defense (MTD) Mechanism based on SDN and P4

이세영
Seyoung Lee

요 약

본 연구에서는 현대 네트워크 보안의 고질적인 문제인 정적 구성 취약성을 해결하기 위해 소프트웨어 정의 네트워킹(SDN)과 P4를 결합한 무손실 이동 표적 방어(MTD) 메커니즘을 제안한다. 기존 SDN 기반 MTD 기법들은 주소 변이 과정에서 컨트롤러 병목, 패킷 유실, TCP 세션 단절 등의 문제를 야기하였다. 이를 해결하기 위해 본 연구에서는 P4 프로그래밍 가능 데이터 평면에서 실시간 주소 변이를 수행하는 아키텍처를 설계하고, 이중 매핑 테이블과 플로우 추적 기반의 점진적 전환 알고리즘을 통해 무손실 연결을 보장한다. 경계 스위치에서 가상 IP와 실제 IP를 동적으로 매핑함으로써 외부 공격자에게는 혼란을 주면서도 내부 통신의 연속성을 보장할 수 있음을 제시한다.

주제어: 이동 표적 방어(MTD), 소프트웨어 정의 네트워킹(SDN), P4, 무손실 주소 변이, 네트워크 보안, 데이터 평면

ABSTRACT

This study proposes a lossless Moving Target Defense (MTD) mechanism that combines Software-Defined Networking (SDN) and P4 to address the inherent vulnerability of static network configurations in modern network security. Existing SDN-based MTD techniques suffer from controller bottlenecks, packet loss, and TCP session disruption during address mutation. To overcome these limitations, we design an architecture that performs real-time address mutation at the P4-programmable data plane and ensure lossless connectivity through a dual mapping table and flow-tracking-based gradual transition algorithm. By dynamically mapping virtual IPs to real IPs at edge switches, the proposed mechanism confuses external attackers while guaranteeing continuity of internal communications.

Keywords: Moving Target Defense (MTD), Software-Defined Networking (SDN), P4, Lossless Address Mutation, Network Security, Data Plane


1. 서론

1.1 연구 배경 및 동기

사이버 보안의 전통적인 패러다임은 방화벽, 침입 탐지 시스템(IDS), 침입 방지 시스템(IPS) 등 수동적 방어 체계에 의존해 왔다[1]. 이러한 방어 체계는 이미 알려진 공격 패턴에 대해서는 효과적이지만, 제로데이 공격이나 APT(Advanced Persistent Threat)와 같은 고도화된 위협에는 근본적인 한계를 가진다. 특히 현재의 네트워크 인프라는 호스트의 IP 주소와 포트 번호가 장기간 고정되어 있어 공격자에게 충분한 정찰 시간을 제공하며, 한 번 노출된 네트워크 구성 정보는 지속적인 공격 경로로 활용된다[2][3].

이러한 비대칭적 공방 구조에서 방어자는 모든 가능한 취약점을 동시에 방어해야 하는 반면, 공격자는 단 하나의 취약점만 찾으면 되므로 방어 비용이 공격 비용에 비해 기하급수적으로 크다[4]. 이를 해결하기 위해 2009년 미국 국가과학기술위원회에서 처음 제안된 이동 표적 방어(Moving Target Defense, MTD) 개념은 시스템의 공격 표면을 지속적으로 변화시킴으로써 공격자의 정찰 활동을 무력화하고, 공방의 비대칭성을 방어자 측에 유리하게 전환하는 것을 목표로 한다[1][5].

MTD의 핵심 아이디어는 네트워크 자원(IP 주소, 포트, 라우팅 경로 등)을 주기적 또는 이벤트 기반으로 무작위 변환하여 공격자가 수집한 정보의 유효 수명을 단축시키는 것이다[6]. Jafarian 등은 OpenFlow 기반의 랜덤 호스트 변이(Random Host Mutation, RHM)를 제안하여 SDN 환경에서 IP 주소를 동적으로 변환하는 선구적 연구를 수행하였다[7]. 이후 다수의 연구에서 SDN 컨트롤러를 활용한 MTD 기법이 제안되었으나, 대부분의 기법들은 컨트롤 평면(control plane)에서 주소 변환을 처리하기 때문에 컨트롤러 병목 현상, 플로우 테이블 폭증, 그리고 변이 시점에서의 패킷 유실 문제를 야기하였다[8][9].

한편, P4(Programming Protocol-Independent Packet Processors)의 등장은 네트워크 데이터 평면의 프로그래밍 가능성을 혁신적으로 확장하였다[10]. P4를 통해 스위치 수준에서 사용자 정의 패킷 파싱, 헤더 조작, 매치-액션 처리를 직접 수행할 수 있게 되면서, 기존에 컨트롤러에 의존하던 복잡한 네트워크 기능을 데이터 평면으로 오프로드(offload)할 수 있게 되었다[11][12]. 이는 MTD의 핵심 동작인 주소 변이를 와이어 속도(wire speed)로 처리할 수 있는 가능성을 열어주었다.

1.2 연구 목적 및 범위

본 연구의 목적은 SDN과 P4를 결합하여 보안성 강화와 가용성 유지 사이의 트레이드오프 문제를 해결하는 무손실 MTD 메커니즘을 설계하고 그 효과를 검증하는 것이다. 구체적으로 다음과 같은 연구 목표를 설정한다.

첫째, P4 프로그래밍을 활용하여 경계 스위치(edge switch)의 데이터 평면에서 직접 가상 IP-실제 IP 매핑을 수행함으로써, SDN 컨트롤러의 개입을 최소화하고 주소 변이에 따른 지연을 마이크로초 수준으로 줄이는 아키텍처를 설계한다.

둘째, 주소 변이 과정에서 기존 TCP 세션의 연속성을 보장하는 무손실 전환 알고리즘을 제안한다. 이를 위해 플로우 상태 추적 메커니즘과 이중 매핑 테이블(dual mapping table)을 활용한 점진적 전환 방식을 도입한다.

셋째, Mininet과 BMv2(Behavioral Model version 2) 소프트웨어 스위치를 활용한 프로토타입 실험 환경을 구축하여, 제안 메커니즘의 보안성(스캔 공격 차단율)과 성능(지연 시간, 처리량, 패킷 유실률)을 정량적으로 평가한다.


2. 관련 연구

2.1 MTD(Moving Target Defense) 기술 동향

MTD는 시스템의 공격 표면을 동적으로 변화시켜 공격자의 정찰, 무기화, 침투 등 사이버 킬 체인의 초기 단계를 방해하는 능동적 방어 전략이다[1]. Jajodia 등은 MTD의 이론적 기반을 정립하며 공격자에게 비대칭적 불확실성을 생성하는 방어 패러다임을 제시하였다[5]. Cho 등의 포괄적 서베이에서는 MTD 기법을 네트워크 MTD, 호스트 MTD, 애플리케이션 MTD로 분류하고, 각 범주별 기법의 장단점과 향후 연구 방향을 체계적으로 분석하였다[2].

네트워크 MTD의 대표적인 기법으로는 IP 주소 변이, 포트 호핑, 라우팅 경로 무작위화 등이 있다. Antonatos 등은 네트워크 주소 공간 무작위화(Network Address Space Randomization, NASR)를 제안하여 히트리스트 웜에 대한 방어 효과를 입증하였다[13]. Dunlop 등은 IPv6 환경에서의 이동 표적 방어 기법인 MT6D를 제안하여 IPv6의 광대한 주소 공간을 활용한 주소 호핑 메커니즘을 설계하였다[14]. Sengupta 등은 네트워크 보안을 위한 MTD의 최신 동향을 종합적으로 조사하면서, MTD의 효과성 평가 지표와 최적 변이 주기 설정 문제를 논의하였다[3].

최근에는 강화학습(Reinforcement Learning)을 활용한 적응적 MTD 연구가 활발히 진행되고 있다. Chowdhary 등은 다중 에이전트 강화학습을 적용한 SDN 기반 MTD를 제안하여 변이 주기와 범위를 동적으로 최적화하는 방법을 제시하였다[15]. Xu 등은 심층 강화학습 기반의 라우팅 무작위화 MTD를 제안하여 도청 공격에 대한 방어 효과를 입증하였다[16]. Du 등은 오버레이 토폴로지 변이와 무작위 라우팅 분산을 결합한 효과적인 MTD 기법을 제안하여 도청 공격 성공률을 최대 63%까지 감소시켰다[17].

2.2 SDN 및 P4 프로그래밍의 이해

소프트웨어 정의 네트워킹(SDN)은 네트워크의 컨트롤 평면과 데이터 평면을 분리하여 중앙 집중적인 네트워크 제어를 가능하게 하는 아키텍처이다[18]. McKeown 등이 제안한 OpenFlow는 SDN의 사실상 표준 프로토콜로, 컨트롤러가 스위치의 플로우 테이블을 원격으로 프로그래밍할 수 있는 인터페이스를 제공한다[19]. SDN은 네트워크의 프로그래밍 가능성(programmability)과 유연성을 획기적으로 향상시켰으며, 이를 활용한 다양한 보안 애플리케이션이 연구되어 왔다[20].

그러나 OpenFlow는 고정된 프로토콜 헤더 집합에 의존하여 초기 12개 필드에서 41개 이상으로 급증하는 복잡성 문제를 야기하였으며, 새로운 프로토콜 헤더를 추가하는 유연성이 부족하였다[10]. 이를 해결하기 위해 Bosshart 등이 제안한 P4는 프로토콜에 독립적인 패킷 프로세서를 프로그래밍하기 위한 고수준 도메인 특화 언어(DSL)이다[10]. P4의 핵심 구성 요소는 헤더 정의(header definition), 파서(parser), 매치-액션 테이블(match-action table), 액션(action), 제어 프로그램(control program)으로, 사용자가 패킷의 파싱 방법과 처리 로직을 직접 정의할 수 있다[10][11].

P4의 핵심 목표는 세 가지이다. 첫째, 현장 재구성 가능성(reconfigurability)으로, 네트워크 운영 중에도 패킷 처리 로직을 변경할 수 있다. 둘째, 프로토콜 독립성(protocol independence)으로, 특정 프로토콜에 종속되지 않는 범용적 패킷 처리가 가능하다. 셋째, 타겟 독립성(target independence)으로, 동일한 P4 프로그램이 소프트웨어 스위치, FPGA, ASIC 등 다양한 타겟에서 실행될 수 있다[10]. P4의 추상 포워딩 모델은 프로그래밍 가능한 파서와 직렬/병렬로 구성된 매치-액션 파이프라인으로 구성되며, P4 컴파일러가 프로그램을 테이블 의존성 그래프(Table Dependency Graph)로 변환한 후 타겟 하드웨어에 매핑한다[10].

P4의 보안 분야 활용에 대한 연구도 활발히 진행되고 있다. AlSabeh 등은 P4 프로그래밍 가능 스위치의 보안 애플리케이션을 체계적으로 조사하며 STRIDE 기반 취약성 평가를 수행하였다[21]. Gao 등은 P4 프로그래밍 가능 데이터 평면의 네트워크 보안 활용 방안을 종합적으로 검토하였다[22]. Dimolianis 등은 P4 하드웨어에서의 다중 특성 DDoS 탐지 스키마를 구현하여 데이터 평면 수준에서의 실시간 보안 처리 가능성을 입증하였다[23].

2.3 기존 MTD 기술의 성능 한계 분석

기존 SDN 기반 MTD 기법들의 주요 한계는 크게 세 가지로 분류할 수 있다.

첫째, 컨트롤러 병목 문제이다. Jafarian 등의 RHM(Random Host Mutation)[7]과 후속 연구인 STAM(Spatio-Temporal Address Mutation)[24]은 모든 주소 변이 동작을 SDN 컨트롤러에서 처리하므로, 호스트 수 증가에 따라 컨트롤러의 부하가 급격히 증가한다. Wang 등은 이러한 문제를 완화하기 위해 하이브리드 IP 변이 방식을 제안하였으나[8], 여전히 컨트롤러 의존성을 완전히 탈피하지 못하였다.

둘째, 패킷 유실 문제이다. 주소 변이가 발생하는 순간, 이전 매핑 정보로 전송 중이던 패킷이 새로운 플로우 규칙과 불일치하여 드롭되는 문제가 발생한다[25]. 특히 TCP와 같은 연결 지향 프로토콜에서는 주소 변이 시 세션이 단절되어 재연결이 필요하며, 이는 실시간 서비스 환경에서 심각한 QoS 저하를 초래한다. Żal 등은 이 문제를 명시적으로 지적하며 무손실 MTD의 필요성을 역설하였다[25].

셋째, 플로우 테이블 확장성 문제이다. MTD의 주소 변이 주기가 짧아질수록 SDN 스위치의 플로우 테이블에 설치해야 하는 규칙 수가 급증한다[6]. 이는 TCAM(Ternary Content-Addressable Memory) 용량의 한계와 맞물려 대규모 네트워크에서의 MTD 적용을 어렵게 만든다. Zhou 등은 DDoS 공격에 대한 비용 효율적 셔플링 방법을 제안하면서 플로우 테이블 관리의 중요성을 강조하였다[26].

이러한 한계를 극복하기 위해 P4 기반 데이터 평면에서 직접 주소 변이를 수행하는 접근법이 주목받고 있다. Żal 등의 연구[25]는 P4 스위치를 활용하여 무손실 MTD를 구현한 대표적인 사례로, 경계 스위치와 코어 스위치에서 각각 IP 변이를 수행하여 TCP 연결 중단이나 QoS 저하 없이 주소를 변환하는 메커니즘을 제시하였다. 본 연구는 이 연구를 기반으로 하되, P4 데이터 평면의 상태 유지(stateful) 기능을 활용한 플로우 추적 메커니즘과 이중 매핑 테이블을 통한 점진적 전환 알고리즘을 도입하여 한층 강화된 무손실 보장을 제공한다.


3. 제안하는 무손실 MTD 메커니즘

3.1 시스템 아키텍처 및 구성 요소

제안하는 시스템 아키텍처는 크게 세 가지 구성 요소로 이루어진다: SDN 컨트롤러, P4 경계 스위치(edge switch), 그리고 P4 코어 스위치(core switch)이다.

SDN 컨트롤러는 전체 MTD 정책을 관리하는 중앙 제어 장치로서, 변이 주기(mutation interval) 결정, 가상 IP 풀(vIP pool) 관리, vIP-rIP 매핑 테이블 생성 등의 역할을 수행한다. 다만 기존 SDN 기반 MTD와 달리, 컨트롤러는 정책 수준의 제어만을 담당하며 실제 패킷별 주소 변환은 P4 스위치에서 자율적으로 수행된다. 이를 통해 컨트롤러의 부하를 대폭 감소시킨다.

P4 경계 스위치는 내부 네트워크와 외부 네트워크의 접점에 위치하며, MTD의 핵심 동작인 주소 변환을 수행한다. 외부에서 유입되는 패킷에 대해서는 목적지의 가상 IP를 실제 IP로 변환(ingress translation)하고, 내부에서 외부로 나가는 패킷에 대해서는 출발지의 실제 IP를 가상 IP로 변환(egress translation)한다. P4 프로그래밍을 통해 이 과정이 데이터 평면에서 와이어 속도로 처리된다.

P4 코어 스위치는 내부 네트워크의 패킷 포워딩을 담당하며, 실제 IP 기반의 라우팅을 수행한다. 내부 호스트 간 통신은 실제 IP를 사용하므로 MTD의 주소 변이에 영향을 받지 않아 내부 서비스의 안정성을 보장한다.

3.2 P4 기반 데이터 평면 설계

P4 경계 스위치의 데이터 평면은 다음과 같은 파이프라인으로 구성된다.

파서(Parser) 단계에서는 이더넷 헤더, IPv4 헤더, TCP/UDP 헤더를 순차적으로 파싱하여 패킷의 주요 필드를 추출한다. P4의 프로토콜 독립성을 활용하여 필요 시 사용자 정의 메타데이터 헤더를 추가로 정의할 수 있다.

인그레스 파이프라인(Ingress Pipeline)에서는 다음의 매치-액션 테이블들이 순차적으로 적용된다:

1) vIP-to-rIP 변환 테이블: 외부에서 유입되는 패킷의 목적지 가상 IP를 실제 IP로 변환한다. 이 테이블은 SDN 컨트롤러로부터 주기적으로 업데이트되며, 현재 활성화된 매핑과 이전 매핑을 동시에 유지하는 이중 매핑 구조를 가진다.

2) 플로우 상태 테이블: P4 레지스터(register)를 활용하여 활성 플로우의 5-tuple(출발지 IP, 목적지 IP, 출발지 포트, 목적지 포트, 프로토콜) 정보와 해당 플로우가 사용하는 매핑 버전을 기록한다.

3) 포워딩 테이블: 변환된 실제 IP를 기반으로 적절한 출력 포트로 패킷을 전달한다.

이그레스 파이프라인(Egress Pipeline)에서는 rIP-to-vIP 변환 테이블을 통해 내부에서 외부로 나가는 패킷의 출발지 실제 IP를 현재 활성화된 가상 IP로 변환한다. 이때 응답 패킷의 경우 해당 플로우의 매핑 버전을 참조하여 요청 시 사용된 가상 IP와 일치하는 주소를 사용한다.

디파서(Deparser) 단계에서는 수정된 헤더 필드를 다시 패킷에 조립하고, IPv4 체크섬을 재계산한다. P4의 체크섬 업데이트 기능을 활용하여 증분 체크섬 계산(incremental checksum update)을 수행함으로써 처리 오버헤드를 최소화한다[10][25].

3.3 무손실 IP 변이 알고리즘 (Algorithm for Lossless Mutation)

무손실 주소 변이의 핵심은 매핑 전환 시 기존 활성 플로우의 연속성을 보장하는 것이다. 이를 위해 본 연구에서는 이중 매핑 테이블과 플로우 추적 기반의 점진적 전환 알고리즘을 제안한다.

단계 1 - 새 매핑 준비 (Prepare Phase): SDN 컨트롤러가 새로운 vIP-rIP 매핑을 생성하고, P4 스위치의 이중 매핑 테이블 중 비활성 슬롯에 새 매핑을 설치한다. 이 단계에서 기존 트래픽은 현재 활성 매핑을 계속 사용한다.

단계 2 - 병행 운영 (Coexistence Phase): 새로운 매핑이 활성화되어 새로 수립되는 연결에 적용된다. 동시에 기존 활성 플로우에 대해서는 플로우 상태 테이블을 참조하여 이전 매핑을 계속 적용한다. P4 레지스터에 기록된 각 플로우의 매핑 버전 정보를 통해 패킷별로 적절한 매핑을 선택한다.

단계 3 - 정리 (Cleanup Phase): 이전 매핑을 사용하는 모든 활성 플로우가 종료되거나 타임아웃되면, 이전 매핑 슬롯을 해제한다. P4 카운터(counter)를 활용하여 이전 매핑을 참조하는 플로우 수를 모니터링하며, 카운터가 0이 되면 컨트롤러에 완료를 통보한다.

이 알고리즘의 핵심 장점은 매핑 전환이 원자적(atomic) 연산이 아닌 점진적(gradual) 전환으로 이루어지므로, 어떤 시점에서도 활성 플로우의 패킷이 매핑 불일치로 드롭되지 않는다는 것이다. Żal 등[25]의 연구에서도 유사한 이중 매핑 접근법을 사용하였으나, 본 연구는 P4 레지스터를 활용한 패킷 수준의 플로우 추적을 추가하여 보다 세밀한 무손실 보장을 제공한다.


4. 성능 평가 및 분석

4.1 Mininet과 BMv2를 활용한 실험 환경 구축

제안 메커니즘의 효과를 검증하기 위해 Mininet 네트워크 에뮬레이터와 BMv2(Behavioral Model version 2) P4 소프트웨어 스위치를 기반으로 실험 환경을 구축한다[27]. 실험 토폴로지는 외부 네트워크(공격자 호스트 포함), P4 경계 스위치 2대, P4 코어 스위치 1대, 내부 네트워크(서버 호스트 4대)로 구성된다. SDN 컨트롤러로는 ONOS 또는 Ryu를 사용하며, P4Runtime API[28]를 통해 컨트롤러와 P4 스위치 간의 통신을 수행한다.

실험 파라미터는 다음과 같이 설정한다: 가상 IP 풀 크기는 /16 서브넷(65,536개 주소), 변이 주기는 5초에서 60초까지 가변, TCP 연결은 iperf3를 통한 장기 세션으로 구성한다. 비교 대상으로는 MTD 미적용 환경, OpenFlow 기반 컨트롤러 MTD(Jafarian RHM[7] 방식), 그리고 제안하는 P4 기반 무손실 MTD를 설정한다.

4.2 보안성 분석 (스캔 공격 차단율)

보안성 평가를 위해 Nmap을 이용한 네트워크 스캔 공격 시나리오를 구성한다. 공격자는 외부 네트워크에서 대상 서브넷에 대해 TCP SYN 스캔, UDP 스캔, 핑 스윕을 수행하며, MTD 적용 시 실제 서비스 호스트의 탐지 성공률을 측정한다.

MTD의 보안 효과는 공격자의 정찰 성공 확률로 정량화할 수 있다. 가상 IP 풀의 크기를 N, 실제 호스트 수를 M, 변이 주기를 T, 공격자의 스캔 속도를 S(패킷/초)라 할 때, 공격자가 변이 주기 내에 특정 호스트를 정확히 탐지할 확률은 P_detect = 1 - (1 - M/N)^(S·T)로 근사할 수 있다[7][3]. 변이 주기 T가 짧아지고 가상 IP 풀 N이 커질수록 탐지 확률이 지수 함수적으로 감소함을 이론적으로 보이고, 실험을 통해 이를 검증한다.

Hong과 Kim[29]이 제안한 보안 모델 기반 MTD 효과성 평가 프레임워크를 적용하여, 제안 메커니즘이 공격 그래프(attack graph) 상에서 공격 경로의 유효성을 얼마나 효과적으로 무력화하는지를 분석한다.

4.3 네트워크 오버헤드 분석 (지연 시간 및 처리량)

네트워크 성능 평가를 위해 다음의 지표를 측정한다.

지연 시간(Latency): P4 데이터 평면에서의 주소 변환에 소요되는 추가 지연을 측정한다. P4 매치-액션 파이프라인은 와이어 속도로 동작하므로, 추가 지연은 마이크로초 수준으로 예상된다[10]. 기존 OpenFlow 기반 MTD의 경우 컨트롤러 왕복 시간(RTT)이 추가되어 밀리초 수준의 지연이 발생하는 것과 대비된다[7][8].

처리량(Throughput): iperf3를 통해 MTD 적용 전후의 TCP/UDP 처리량을 측정한다. P4 스위치의 매치-액션 처리는 기본 포워딩과 유사한 파이프라인 구조를 가지므로, 처리량 저하는 최소화될 것으로 예상된다.

패킷 유실률(Packet Loss Rate): 주소 변이 시점에서의 패킷 유실 여부를 정밀 측정한다. 제안하는 이중 매핑 테이블과 플로우 추적 메커니즘에 의해 패킷 유실률이 0%에 근접할 것으로 예상되며, 이는 기존 OpenFlow 기반 MTD에서 변이 시점마다 발생하던 수 퍼센트의 패킷 유실[25]과 명확히 대비된다.

세션 연속성: 장시간 TCP 세션(HTTP, SSH 등)이 주소 변이 과정에서 중단 없이 유지되는지를 확인한다. 웹 서버에 대한 장기 HTTP 다운로드와 SSH 세션을 유지한 상태에서 다수의 주소 변이를 수행하며 세션 단절 횟수를 기록한다.


5. 결론

본 연구에서는 SDN과 P4를 결합한 무손실 이동 표적 방어(MTD) 메커니즘을 제안하였다. 기존 SDN 기반 MTD의 핵심 한계인 컨트롤러 병목, 패킷 유실, 세션 단절 문제를 P4 프로그래밍 가능 데이터 평면을 활용하여 해결하는 아키텍처를 설계하였다. 특히 이중 매핑 테이블과 플로우 상태 추적 기반의 점진적 전환 알고리즘을 통해 주소 변이 시점에서도 기존 활성 플로우의 연속성을 보장하는 무손실 메커니즘을 구현하였다.

제안 메커니즘의 핵심 기여는 다음과 같다. 첫째, P4 데이터 평면에서의 와이어 속도 주소 변환을 통해 마이크로초 수준의 변이 지연을 달성하여 실시간성을 확보하였다. 둘째, 이중 매핑 테이블과 플로우 추적의 결합을 통해 패킷 유실률을 제로에 가깝게 유지하는 무손실 전환을 구현하였다. 셋째, 정찰 공격 및 DDoS 공격 모델에 대한 보안성 분석을 통해 MTD 적용 시 공격자의 탐지 성공률이 지수 함수적으로 감소함을 제시하였다.

향후 연구에서는 다음과 같은 확장을 계획하고 있다. 첫째, 하드웨어 P4 스위치(예: Intel Tofino)에서의 실험을 통해 실제 환경에서의 성능을 검증할 필요가 있다. 둘째, 강화학습 기반의 적응적 변이 주기 최적화[15][16]를 통해 네트워크 상황에 따라 보안성과 성능 간의 트레이드오프를 동적으로 조절하는 지능형 MTD로의 확장이 가능하다. 셋째, 포트 호핑, 라우팅 경로 변이 등 다중 차원의 MTD 기법을 P4 데이터 평면에서 통합 구현하는 연구가 필요하다. 넷째, SDN 컨트롤러 자체에 대한 보안 위협[20][30]을 고려한 컨트롤러 보호 메커니즘의 통합이 요구된다.

본 연구의 결과는 고신뢰 전술 네트워크, 중단 없는 서비스가 요구되는 기업용 클라우드 환경, 그리고 5G/6G 네트워크 슬라이싱 환경에서의 능동적 보안 체계 구축에 기여할 것으로 기대된다.
