# P4 프로그래머블 네트워크를 통한 정책기반 인-네트워크 보안 관리 방법

## Abstract

본 논문은 네트워크 데이터 평면 프로그래밍 언어인 P4를 활용하여, 네트워크 스위치 단에서 직접 보안 공격을 탐지하고 대응할 수 있는 인-네트워크(In-Network) 보안 관리 방법을 제안합니다. 기존에는 방화벽이나 IPS 같은 별도 보안 장비에 의존해야 했고, OpenFlow 기반 SDN 스위치에서는 플로우 상태 모니터링이 안 돼서 동적인 공격에 즉각 대응하기 어려웠습니다. 이 논문에서는 P4 스위치에서 모든 네트워크 플로우를 실시간으로 모니터링하면서, 분산 DDoS 공격이나 IP 스푸핑 공격 같은 보안 위협을 스위치 단에서 바로 처리하는 방법을 제시합니다. 거기에 SDN 제어기를 통해 보안 정책을 동적으로 업데이트할 수 있게 해서, 다양한 네트워크 환경에서의 보안 요구사항도 반영할 수 있도록 하였습니다.

## Why

- **기존 보안 체계의 한계:** 네트워크 스위치는 고정된 룰 기반의 방화벽이나 ACL로만 보안을 처리할 수 있어서, 동적으로 바뀌는 사이버 공격에 능동적으로 대처하지 못합니다. 별도의 보안 장비를 추가해야 하는데, 그래도 10G/100G 이상의 라인 속도에서 실시간으로 처리하기는 어렵습니다.
- **SDN 제어기 기반 보안의 문제:** OpenFlow 기반 SDN 스위치에서는 보안 위협 패킷이 들어오면 SDN 제어기한테 물어봐야 하는데, 이 과정에서 물리적인 통신 지연이 발생해서 즉각적인 대응이 안 됩니다. 더 심각한 건 SDN 제어기 자체가 DDoS 공격의 대상이 될 수도 있다는 점입니다.
- **데이터 평면 프로그래밍의 가능성:** P4와 Tofino 같은 프로그래밍 가능한 고성능 스위치 칩이 나오면서, 스위치 단에서 직접 패킷을 모니터링하고 차단하는 것이 기술적으로 가능해졌습니다.

## What

- 연구 질문: P4 데이터 평면 프로그래밍을 통해 네트워크 스위치 단에서 직접 보안 공격을 탐지하고 차단할 수 있는가? 그리고 SDN 제어기와 연동하여 정책 기반으로 동적인 보안 관리가 가능한가?
- 연구 목적:
  1. P4 스위치에서 네트워크 플로우를 실시간 모니터링하면서 분산 DDoS 및 스푸핑 공격을 탐지·차단하는 인-네트워크 보안 시나리오 제안
  2. SDN 제어기를 통해 보안 정책(블랙리스트/화이트리스트)을 동적으로 반영할 수 있는 정책 기반 보안 관리 구조 제시

## How

### 보안 시나리오
- **분산 DDoS 공격 차단:** P4 스위치에서 모든 네트워크 세션의 상태를 모니터링합니다. 유효한 TCP SYN 정보가 없는 패킷이 들어오면 해당 플로우의 카운터를 증가시키고, 특정 임계치를 초과하면 DDoS 공격으로 판단해서 스위치 단에서 바로 차단하거나 보안 장비로 리다이렉션합니다. 출발지 IP/포트, 목적지 IP/포트 정보를 SRAM의 매치/액션 테이블 엔트리로 관리합니다.
- **스푸핑 공격 차단:** DHCP 스푸핑, IP 스푸핑, ARP 스푸핑 등에 대해 정상 사용자의 IP/MAC 정보를 화이트리스트 기반 테이블로 유지합니다. 화이트리스트에 없는 IP/MAC 조합이 발견되면 P4 스위치에서 즉각 차단합니다.

### 정책 기반 관리 구조
- SDN 제어기는 보안 정책을 반영하는 역할만 수행하고, 실제 패킷별 처리는 P4 스위치가 자율적으로 합니다.
- SDN 제어기를 통해 블랙리스트/화이트리스트를 업데이트해서 P4 스위치를 항상 최신 보안 정책으로 운영합니다.
- P4 스위치 내 SRAM 레지스터를 통해 패킷 당 라인 속도로 플로우 정보를 모니터링하고, 탐지된 공격 정보를 SDN 제어기에 피드백해서 다른 P4 스위치에도 전파하는 피드백 기반 보안 운영 환경을 구현합니다.

### 보안 프로시저 순서
입력 패킷 파싱 → 매치/액션 테이블 적용 → 블랙리스트 기반 선제 차단 → 화이트리스트 기반 허용 → DDoS 공격 대응 → 스푸핑 공격 대응 → 출력 패킷 디파싱 순서로 처리됩니다.

## Contribution

- **네트워크 단에서의 즉각적인 보안 대응:** 기존에는 별도 보안 장비가 필요했던 DDoS 탐지·차단을 P4 스위치 단에서 직접 처리할 수 있게 함으로써, 10G/100G 이상의 라인 속도에서도 보안 처리가 가능하다는 점을 보여줬습니다.
- **정책 기반 보안 관리 체계:** SDN 제어기와 P4 스위치를 연동해서 보안 정책을 동적으로 반영할 수 있는 구조를 제안했습니다. 블랙리스트/화이트리스트를 실시간 업데이트하면서 피드백 기반의 보안 운영이 가능합니다.
- **적용 범위의 확장:** 기존 BYOD 환경의 호스트 네트워크 수준을 넘어서, 백본 및 액세스 구간 모두에 적용 가능한 인-네트워크 보안 방법을 제시했다는 점에서 차별화됩니다.

## Critic

이 논문은 P4를 보안에 활용할 수 있다는 방향성을 잘 제시해줬고, 제 연구에서 P4를 선택하게 된 가장 큰 계기가 된 논문입니다. 다만 읽으면서 몇 가지 아쉬운 점도 있었습니다.

- **실험적 검증 부족:** DDoS 차단이나 스푸핑 차단 시나리오를 제안하고 있지만, 실제로 P4 스위치에서 구현하고 성능을 측정한 실험 결과가 논문에 포함되어 있지 않습니다. 제안 방법의 실효성을 주장하기 위해서는 실제 구현 및 성능 평가가 필요해 보입니다.
- **SRAM 용량 한계에 대한 구체적 해결책 부족:** 논문 자체에서도 Tofino 칩의 SRAM이 O(10MB) 수준이라 수백만 개의 플로우를 다 저장하기 어렵다고 지적하고 있는데, TEA 같은 외부 DRAM 활용 방안을 언급만 하고 구체적인 적용 방법은 제시하지 않았습니다. 대규모 네트워크에서는 이 메모리 문제가 꽤 심각할 것 같습니다.
- **DDoS 탐지 정확도에 대한 논의 부족:** TCP SYN Flood 패턴을 기반으로 카운터가 임계치를 초과하면 공격으로 판단하는 방식인데, 정상 트래픽이 급증하는 상황(예를 들어 대규모 이벤트)에서 오탐이 발생할 가능성에 대한 고려가 없었습니다. 임계치를 어떻게 설정하느냐에 따라 탐지율과 오탐률이 크게 달라질 텐데, 이 부분에 대한 분석이 있었으면 좋겠다는 생각이 들었습니다.
