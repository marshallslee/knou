# P4 프로그래머블 네트워크를 통한 정책 기반 인-네트워크 보안 관리 방법 (2020)

## Abstract

P4 데이터 평면 프로그래밍 언어를 활용하여 네트워크 내 모든 플로우를 P4 스위치 단에서 실시간으로 모니터링하고, 분산 DDoS 공격 및 IP Spoofing 공격 등의 보안 위협 패킷을 스위치 단에서 직접 처리하는 인-네트워크(In-Network) 보안 관리 방법을 제안한다. SDN 제어기를 통해 사용자 또는 보안 관리자의 운영 정책을 P4 스위치에 반영함으로써, 다양한 네트워크 응용 환경에서의 보안 요구 사항을 동적으로 적용할 수 있다.

## Why

기존 네트워크 스위치는 고정된 룰에 의한 방화벽이나 접근 제어만 가능하여, 보안 위협에 대한 능동적 대응이 극히 제한적이다. OpenFlow 기반 SDN 스위치도 보안 위협 패킷이 유입되면 SDN 제어기에 처리 방법을 요청하고 결과를 다시 받아와야 하므로, 물리적인 통신 지연이 발생하여 즉각적인 대응이 불가능하며 라인 속도(Line-rate)로 공격 패킷을 처리할 수 없다. 또한 SDN 제어기 자체가 DDoS 등의 플러딩 공격에 노출될 수 있고, 기존 SDN 스위치에서는 플로우/세션의 상태 모니터링이 불가하여 동적으로 전개되는 사이버 공격에 대응하기 어렵다.

## What

본 논문은 크게 두 가지를 제안한다:

1. **P4 기반 인-네트워크 보안 시나리오**: P4 스위치 단에서 직접 보안 공격을 탐지하고 대응하는 구체적인 시나리오를 제시한다.
   - **분산 DDoS 공격 차단**: 모든 네트워크 세션의 상태를 모니터링하면서 TCP SYN Flood 패턴을 탐지하고, 임계치 초과 시 스위치 단에서 즉각 차단하거나 보안 장비로 리다이렉션한다. 매치/액션 테이블에 출발지/목적지 IP·Port 정보를 엔트리로 관리하며, 최근 엔트리와 신규 플로우를 비교하여 판단한다.
   - **스푸핑 공격 차단**: DHCP/ARP/IP 스푸핑에 대해 정상 사용자의 IP-MAC 정보를 화이트리스트 테이블로 유지하고, 위장된 패킷을 입력(ingress) 단에서 필터링하여 차단한다.

2. **정책 기반 인-네트워크 보안 관리 구조**: SDN 제어기를 통해 보안 정책(블랙리스트/화이트리스트)을 P4 스위치에 동적으로 반영하는 프레임워크를 제시한다. P4 스위치의 SRAM 레지스터로 플로우 정보를 모니터링하고, 매치/액션 테이블의 키(출발지/목적지 IP·Port, 프로토콜)와 값(차단=0, 허용=1, 리다이렉션=2)을 SDN 제어기가 업데이트함으로써 동적 대응이 가능하다.

## How

인-네트워크 보안 관리 프로시저는 다음 순서로 동작한다:

1. **입력 패킷 파싱** (Ingress Packet Parsing)
2. **P4 매치/액션 테이블 적용**
3. **블랙리스트 기반 차단**: 이미 알려진 공격 IP를 선제적으로 차단하여 네트워크 혼잡 최소화
4. **화이트리스트 기반 포워딩**: 인증된 사용자의 대용량 플로우를 선제적으로 허용하여 불필요한 처리 자원 소모 방지
5. **분산 DDoS 대응**: 유효하지 않은 TCP SYN 패킷의 카운터를 증가시키고, 임계치 초과 시 차단
6. **스푸핑 대응**: IP/MAC 정보가 화이트리스트와 불일치하면 차단
7. **출력 패킷 디파싱** (Egress Packet Deparsing)

이 프로시저는 P4 언어로 작성·컴파일하여 P4 스위치에 로딩한다. 또한 P4 스위치에서 탐지한 공격 정보를 SDN 제어기로 전달하고, SDN 제어기가 이를 네트워크 내 다른 P4 스위치에 전파하는 피드백 기반의 보안 운영 환경(Closed Operation Loop)을 구현한다.

SRAM과 TCAM을 활용하여 매치/액션 테이블을 구현하며, SRAM 레지스터는 패킷 당 라인 속도로 동작하여 실시간 모니터링을 가능하게 한다.

## Contribution

- **데이터 평면에서의 즉각적 보안 대응**: 기존에 SDN 제어기를 거쳐야 했던 보안 처리를 P4 스위치 단에서 라인 속도(10G/100G 이상)로 수행할 수 있게 했다.
- **종단간 적용 가능성**: BYOD 환경의 호스트 단에 국한되었던 기존 연구와 달리, 백본 네트워크부터 캠퍼스 네트워크, 호스트까지 종단간 적용이 가능한 구조를 제시했다.
- **정책 기반 동적 보안 관리**: SDN 제어기를 통해 블랙리스트/화이트리스트를 동적으로 업데이트하여 다양한 보안 요구 사항을 반영할 수 있는 메커니즘을 제안했다.
- **피드백 기반 운영 환경**: P4 스위치의 탐지 정보 → SDN 제어기 → 타 스위치 전파로 이어지는 Closed Operation Loop를 통해 네트워크 전체의 보안 상태를 실시간으로 동기화할 수 있다.

## Critic

본 논문은 P4 기반 인-네트워크 보안의 개념과 아키텍처를 제시하는 데 초점을 맞추고 있으나, 실제 구현 및 성능 평가 결과가 포함되어 있지 않다. 분산 DDoS 탐지에 사용되는 TCP 포트 카운트 임계치 기반 방식의 정확도, 오탐률(false positive) 등에 대한 정량적 분석이 없어서 실제 운영 환경에서의 효과를 가늠하기 어렵다.

또한 5.2절에서 저자 스스로도 인정한 것처럼, P4 기반 Tofino 칩의 SRAM이 O(10MB) 수준이라 수백만 개의 플로우 정보를 저장하는 것이 불가능하다는 한계가 있다. 대규모 네트워크에서 모든 플로우를 모니터링하려면 메모리 부족 문제가 필연적으로 발생하며, TEA 같은 외부 DRAM 활용 방안을 언급하고 있으나 이에 대한 구체적인 구현이나 성능 영향 분석은 제시하지 않았다.

DDoS 탐지 시나리오에서 DPI 및 시퀀싱 번호를 활용한다고 했는데, P4 스위치에서 DPI 수준의 페이로드 검사를 라인 속도로 수행하는 것은 현실적으로 매우 어려운 과제이며, 이 부분에 대한 실현 가능성 검토가 부족하다.

그럼에도 불구하고, 본 논문은 내 연구에서 P4를 선택하게 된 핵심 근거가 되었다. 기존 OpenFlow 기반 SDN 스위치에서는 플로우 상태 모니터링 자체가 불가능하여 동적 사이버 공격에 대응하기 어려웠다는 점, 그리고 P4를 통해 데이터 평면에서 직접 보안 기능을 구현할 수 있다는 가능성을 구체적으로 보여주었다는 점에서 의의가 크다. 내 연구의 무손실 MTD 메커니즘도 이 논문의 P4 기반 매치/액션 파이프라인 구조와 SRAM 레지스터 활용 방식을 참고하여 설계하였다.
